About availability
basics
Module Type Scheduler Remote
report Yes No
check output format When used with the report check module, the output provided to the module will be three columns: 
     percentavailable successfulchecks totalchecks

Dependencies

  • None

Arguments

None.

Description

The availability module determines the percentage of checks that succeeded in the provided data.

The availability module utilizes check records.

Examples

  1. Example reporting output: 
         ping@host1: 100% (88/88)
     ping@host2: 99% (198/199)
  2. Example check formatted output: 
         99 198 199
