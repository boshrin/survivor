About message
basics
Module Type Scheduler Remote
check Yes No
syntax 
     module message {
       message <string>
     }

Dependencies

  • None

Arguments

Name Type Value
message string Message to display (exit with) upon execution.

Description

The message module exits with MODEXEC_NOTICE, using the provided argument as the comment.

Examples

  1. To generate a test message: 
         module message {
       message This is a test
     }
