About full
basics
Module Type Scheduler Remote
format Yes No
syntax 
     format as full

Dependencies

  • None

Arguments

None.

Description

The full module generates a moderately verbose alert message, containing as much detail about a problem (or problem resolution) as possible, including fix status if appropriate. Additionally, if a help file is available, its contents will be included in the message.

Examples

  1. Example full output: 
         Notifying alice@pager.somedomain.org,bob@pager.somedomain.org via sysadmins calllist(s)

     Summary: Test is not responding.
     
      A fix was attempted for this problem.
      Fix Summary:          Foo started
      Fix Return Code:      0

     Host:          bradybunch
     Service:       test
     Time:          18:31:30 02/14/2003
     ReturnCode:    1 (3 instances)
     Token:         Q1232347791

     Test is the single most important application our site runs.
     Drop everything and fix it.  Some hints are in ~bob/tmp.
 
     This message is one-way and may not be replied to.
