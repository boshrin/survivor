About load
basics
Module Type Scheduler Remote
check Yes Yes
syntax 
     module load {
       [warn <number>]
       [prob <number>]
       [warn5 <number>]
       [prob5 <number>]
       [warn15 <number>]
       [prob15 <number>]
     }
     [via <transport>]

Dependencies

  • Executables
    • Linux distributions lacking the rup command may need to install the rup (or rstat-client) package. Failing that, this module may still be used as a remote module.

Arguments

Name Type Value
warn optional number between zero and infinity Minimum one minute load average required to generate a warning.
prob optional number between zero and infinity Minimum one minute load average required to generate a problem.
warn5 optional number between zero and infinity Minimum five minute load average required to generate a warning.
prob5 optional number between zero and infinity Minimum five minute load average required to generate a problem.
warn15 optional number between zero and infinity Minimum fifteen minute load average required to generate a warning.
prob15 optional number between zero and infinity Minimum fifteen minute load average required to generate a problem.

At least one warn or prob value must be provided. A value of zero will prevent a warning or problem from being generated for that test.

Description

The load module determines the one, five, and/or fifteen minute load average on the host to be checked. In order to determine the load, the module executes rup when used as a scheduler module and uptime when used as a remote module.

Examples

  1. To generate a warning when the one minute load exceeds 10 or a problem when the one minute load exceeds 20: 
         module load {
       warn 10
       prob 20
     }
